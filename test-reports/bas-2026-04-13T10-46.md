## Recommendations

1. **Test Data (Medium):** Use unique invoice numbers AND matching FUNC NO per test run. The BAS import matches on both FUNC NO (payment number) and SOURCE DOC NUMBER (invoice number). Both must be updated per run. Add this as a mandatory pre-condition to TC-08 in the test plan.

2. **Documentation (Medium):** Document the BAS Excel column contract in the test plan: FUNC NO = ITS payment number (numeric portion of PAY\#\#\#\#), SOURCE DOC NUMBER = ITS invoice number. This was undocumented and caused multiple failed import attempts across two test sessions.

3. **UI/UX (Low):** Resolve the mode badge display bug ("Live" shown when mode is "Latest"). Misleads testers checking badge to confirm mode.
