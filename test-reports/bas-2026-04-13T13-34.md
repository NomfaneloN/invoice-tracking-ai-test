## Recommendations

1. **Process Observation (TC-08):** The CAPTURED Excel creation required manual Node.js scripting because no Python runtime is available. Consider documenting a repeatable method for generating CAPTURED Excel files for E2E test runs, or pre-generating a template that only needs SOURCE DOC NUMBER and run-specific fields updated.

2. **Process Observation (TC-09):** The payment stub text file requires careful column alignment when changing SOURCE DOC NUMBER length. A stub template generator (or at least a documented column-position reference) would reduce alignment risk.

3. **Workflow Observation:** TC-10 required direct URL navigation to the workflow action because the Gwenb inbox sidebar dropdown intercepted the click on the table row. This is a minor usability issue — the expanded sidebar menu overlays the table and prevents clicking items. Users would need to collapse the sidebar first.
