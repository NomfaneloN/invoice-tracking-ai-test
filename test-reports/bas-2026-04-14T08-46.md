## New Findings

1. **Business query step name is "Resolve Queries"** (not "Manage Supplier Related Queries"): The TC-05c exception path routes to a step named **"Resolve Queries"**, distinct from the supplier query path (TC-11) which routes to "Manage Supplier Related Queries". The Resolve Queries step uses a single confirmation checkbox: "I confirm that all queries have been resolved and necessary information has been provided."

2. **Loop-back behaviour confirmed for business query**: After TC-12 Resolve Queries submit, "Prepare Voucher" reloads inline with all Outcome and Business Unit Response selections reset — same loop-back behaviour as the supplier query path. The workflow then proceeds identically to the happy path from that point.

---

## Recommendations

1. **Dialog typo fix (Medium):** Fix the "Send for business related query" confirmation dialog hint text to correctly read "You selected 'Send for business related query'. Please write a query." — the typo ("relatd") should be corrected. This applies to both supplier and business query dialog variants.
2. **Step name documentation:** Document the distinction between "Resolve Queries" (business query path) and "Manage Supplier Related Queries" (supplier query path) in user guides. The different step names may confuse users expecting a consistent naming convention.
3. **My Items vs Inbox:** Clarify or unify the distinction between "My Items" and "Inbox". Capture Filing steps consistently appear only in Inbox across all runs — either fix My Items to include them or update documentation.
4. **Payment Number display:** Investigate the "Payment Number" field on the Capture Filing form. The value "Yapp679" does not represent the BAS payment number (5045). If it is a different field, the label should be more descriptive; if it should show the stub payment number, the data binding appears incorrect.
