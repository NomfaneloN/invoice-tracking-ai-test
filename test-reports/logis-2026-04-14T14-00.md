## Summary

TC-01 (Register and Upload Invoice) and TC-02 (Certify Invoice) passed. The LOGIS record PAY5087/2026 was created and certified successfully. However, the workflow stalled at TC-03 — the record is not appearing in any user's inbox after TaniaSmith certified the invoice. The most likely cause is that TC-01 was executed by admin rather than Moshadim (as specified in the test plan), disrupting the role-based routing logic.

TC-03 through TC-12 are skipped as a consequence of this blocker.

## Recommendations

1. **Re-run TC-01 as Moshadim (High):** The test plan specifies Moshadim (SCM) as the TC-01 actor. Using admin breaks the routing chain. Re-run starting from TC-01 logged in as Moshadim to establish correct workflow routing before proceeding to TC-03.

2. **Fix Order entity picker search (Medium):** Typing an order number in the search box and pressing Enter returns "No Data". The component appears not to fire a search on Enter key press — only the search icon click triggers a correct search. The workaround required JavaScript intervention. This should work with standard keyboard input.

3. **Investigate invoice display on detail view (Low):** The Invoices section on the workflow detail page shows "No Data" despite a valid invoice row being present. Verify whether this is a rendering delay or a data-binding issue between the workflow action form and the record detail view.

4. **Fix "Live" badge (Low):** The mode badge consistently shows "Live" when mode is "Latest". Should reflect the actual mode.

5. **Document routing dependency on creator role:** If LOGIS workflow routing is role-sensitive (i.e., the BU Supervisor resolved from the SCM who initiated the request), this should be documented clearly in the test plan and/or in the app's known behaviours section, to prevent silent routing failures when tests are run as admin.
