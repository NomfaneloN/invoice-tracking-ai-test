## TC-03: Attempt to submit BAS Request with missing required fields
**Type:** Negative  
**Result:** FAIL

### Assertions
- [!] Validation error message(s) visible — **PARTIAL**: Submit button is disabled when required fields are empty. An inline warning "No Supplier Details. Select an Order" appears. No field-level inline error messages are shown (e.g., no red highlight on Supplier Name or Invoices fields).
- [!] Form does not close / record not saved — Form did not auto-close (PASS), but a Draft record (PAY4703/2026) was **automatically created** the moment the "Create New → BAS Request For Payment" action was initiated, before any user input. The record persists in Drafts even after closing the form.
- [!] No new record appears in the list — PAY4703/2026 appears in the **Drafts** view (created 02/04/2026 11:19). It does not appear in My Items (only records past Draft appear there), but the record does exist in the system.

### Defect: Draft record created before user input
**Severity:** Medium  
**Description:** When a user initiates "Create New → BAS Request For Payment", the application immediately creates and persists a Draft record (with a Ref No assigned) before any form fields are filled. If the user closes the form without submitting, the empty Draft record is retained in the Drafts section.  
**Expected:** No record should be created or persisted until the user submits a valid form.  
**Actual:** PAY4703/2026 was created at 11:19 and visible in Drafts after closing without submitting.  
**Impact:** Over time, the Drafts list accumulates empty abandoned Draft records with no supplier or invoice data.

## Issues Found

| # | Severity | Description |
|---|----------|-------------|
| 1 | Low | Invoice Attachment is a required field not documented in the test plan. The test plan's input data table should include it. |
| 2 | Medium | Draft record is created immediately on form open, before any user input. Closing without submitting leaves an orphaned Draft in the system. |
| 3 | Low | TC-03 negative test: Submit button is disabled (no field-level error messages). Users may not receive clear feedback about *which* fields are missing. |

---

## Recommendations

1. **Update test plan TC-01 input data** to include Invoice Attachment as a required field with a test file path.
2. **Raise a defect** for the auto-creation of Draft records on form open. Consider implementing lazy creation (only persist on first Save or Submit) or add a "Discard Draft" cleanup mechanism.
3. **Enhance TC-03** to explicitly assert that the Submit button is disabled (which is the actual validation mechanism), and note that field-level error messages are not shown — assess whether this meets UX requirements.
