## Recommendations

1. **Dialog hint text bug (High):** Fix the "Send for supplier related query" confirmation dialog hint to correctly read "You selected 'Send for supplier related query'. Please write a query." This label mismatch is confusing and could cause users to second-guess their selection. The typo ("relatd") should also be corrected.
2. **Loop-back behaviour documentation:** The post-TC-11 loop-back to "Prepare Voucher" with all selections reset is expected but may surprise users. Consider adding a brief informational message (e.g. "Supplier query resolved — please complete the voucher preparation again") to clarify why the form re-appears blank.
3. **My Items vs Inbox:** Clarify or unify the distinction between "My Items" and "Inbox". If Capture Filing steps only appear in Inbox, ensure documentation and user guides reflect this, or make them visible in both views.
4. **Payment Number display:** Investigate the "Payment Number" field on the Capture Filing form. If it is intended to show the BAS PAYMENT NUMBER from the stub, it should reflect `5023`, not an unrelated code (`Yapp679`). If it is a different field, the label should be more descriptive.
