## Recommendations

1. **Fix mode badge display** — The "Live" badge shown when mode is "Latest" is misleading to users and could cause confusion about the active configuration. The badge should reflect the actual mode from localStorage.
2. **Resolve "Period In Possession" spinner** — This column never loads a value during the session, suggesting a persistent API error or data unavailability. Investigate the underlying API call.
3. **Consider in-app BAS file editor** — Currently the BAS Payment Register Excel must be manually prepared outside the system before upload. An in-app template or guided data entry form would reduce manual error risk.
4. **Add import feedback messaging** — After clicking Import on BAS Report Import or Payment Stubs Import, the form simply clears with no visible success/error toast. A clear success notification would improve usability.
5. **Document payment number field** — TC-09 requires knowing the PAYMENT NUMBER for the stub file. This value is not currently visible on the workflow action page and had to be inferred from the template. Exposing this on the record view would reduce dependency on offline data.
